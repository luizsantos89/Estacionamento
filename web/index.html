Padrões de Projeto utilizados:

Factory Method: Classe principal: Vaga. Classes que são criadas de acordo com a seleção do usuário: VagaCarro e VagaMoto;

Strategy: No Model do MVC, classe VO, existem as classes para cálculo do valor a ser pago de acordo com o tipo de veículo (carro ou moto) e com o plano do cliente (mensalista ou horista);

Command: No Controller do MVC, existem as classes que especificam as formas de pagamento.

Padrão MVC: Organizando as classes e páginas referentes ao View (Páginas onde o usuário faz a interação com o sistema (as solicitações), Model onde estãos as classes que instanciam os objetos a serem manipulados e o Controller onde estão as regras de negócio do sistema.

Memento: Implementado no Model do MVC, mantém um log de cadastros de clientes, para que seja possível uma correção em caso de erro de digitação ou retorno ao estado anterior em caso de erro sistêmico.

Singleton: Criação do objeto usuario (o usuário que cadastra clientes, vagas e registra pagamentos). O usuário é o mesmo que fica instanciado até que seja deslogado do sistema. Obs: Uma única instância, realizada apenas uma vez, que fará parte de todos os processos até que seja encerrada.
